# Design Contract

Create this contract before implementation or repair intervention. Make every field observable enough to guide a browser check.

```text
Entry mode:
Target user and context:
Desired first impression:
Visual constraints:
Information constraints:
Operation constraints:
State constraints:
Environment constraints:
Primary journey:
Observable completion criteria:
```

## Intake by entry mode

### Reference-led

Extract composition, focal hierarchy, typography, color/material, depth, motion, and operation placement from the supplied reference. Translate observations into constraints and completion criteria; do not copy unrelated product or business content from the reference.

### Brief-led

Establish composition, focal hierarchy, typography, color/material, depth, motion, and operation placement autonomously. Present alternatives only when their trade-offs materially change implementation; otherwise select a coherent direction and record it in the contract.

### Repair-led

Capture the current browser baseline, then define the target defect and acceptance criteria. Preserve the existing visual system unless redesign is explicitly requested.
